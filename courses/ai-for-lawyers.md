## 提示词技巧

### 什么是提示词？

提示词（Prompt）就是你给 AI 的"指令"。好的提示词能让 AI 像一个经验丰富的助手一样工作，差的提示词则让 AI 给出泛泛而谈的答案。

### 六大核心技巧

#### 1. 角色扮演（Role Prompting）

让 AI 扮演特定角色，能显著提升回答的专业性。

**基础版：**
```
分析以下合同条款的风险。
```

**加了角色：**
```
你是一名在顶级律师事务所工作了15年的资深商事律师，
专注于合同风险审查。请以你的专业视角分析以下条款的风险。
```

#### 2. 分步骤指令（Step-by-Step）

把复杂任务分解为明确步骤，避免 AI 遗漏关键环节。

```
请按以下步骤分析这份劳动合同：
第一步：识别所有对劳动者不利的条款
第二步：对每条按风险高低分级（高/中/低）
第三步：针对每条给出修改建议
第四步：给出总体评价和谈判建议
```

#### 3. 给出示例（Few-shot）

告诉 AI 你期望的输出格式，最简单的方式是给一个例子。

```
请按以下格式分析合同条款：

【条款原文】第X条：……
【风险等级】🔴 高风险 / 🟡 中风险 / 🟢 低风险
【风险说明】……
【修改建议】……

现在请用这个格式分析以下条款：……
```

#### 4. 限定输出格式（Output Format）

明确告诉 AI 你想要什么格式：表格、列表、段落还是代码。

```
请将分析结果以 Markdown 表格形式输出，列包括：
条款编号 | 条款摘要 | 风险等级 | 修改建议
```

#### 5. 迭代优化（Iteration）

AI 的第一次回答不够好时，不要重新开始，而是在当前对话中继续优化。

```
[AI 给出了分析]
你的分析太笼统了，请针对"违约金上限"这一点展开，
并引用相关法律条文支持你的判断。
```

#### 6. 思维链（Chain of Thought）

要求 AI 展示推理过程，能提升复杂法律分析的准确性。

```
请一步步分析这个问题：
某公司未经员工同意录制了工作对话，请问这是否违反《个人信息保护法》？
请先分析适用的法律条文，再逐一检查构成要件，最后得出结论。
```

## 合同审查实战

### AI 合同审查的正确姿势

AI 在合同审查中的正确定位是：**初筛 + 分析辅助**，而非替代律师判断。

#### 实战工作流

```
1. 上传合同 → 2. 告知委托方身份 → 3. AI 生成风险清单
→ 4. 律师复核重点条款 → 5. AI 起草修改建议 → 6. 律师最终审定
```

#### 处理长合同的技巧

Claude 的上下文窗口约为 200K tokens，但对于超长合同，建议分章节处理：

```
这是一份100页的股权收购协议，我将分3次提交各部分，
请先记住以下信息后等待我发送合同内容：
- 我方是买方（收购方）
- 本次收购目标为XX公司60%股权
- 重点关注：价格调整机制、陈述与保证、赔偿条款

明白了吗？请在我发送第一部分后开始分析。
```

#### 风险条款识别清单

以下是常见高风险条款类型，可直接告诉 AI 重点关注：

**商业合同通用：**
- 违约金设置（金额、触发条件、是否可叠加）
- 责任限制条款（尤其是间接损失排除）
- 不可抗力定义（范围是否过宽）
- 管辖与争议解决条款
- 合同变更和终止条件

**采购/服务合同：**
- 付款条件和延迟付款的处理
- 验收标准和争议处理
- 知识产权归属
- 保密义务范围

**劳动合同：**
- 竞业限制（范围、期限、补偿金）
- 保密条款（范围是否过宽）
- 工资组成与绩效考核机制

## 研究工作流

### AI 辅助法律研究的四层应用

#### 第一层：文献检索与筛选
```
我正在研究"数据可携权在中国法的本土化实践"，
请帮我：
1. 列出这个主题的核心关键词（中英文）
2. 推荐5个最相关的数据库检索策略
3. 识别这个领域的标志性文献和核心学者
```

#### 第二层：文献阅读与综述
```
我上传了3篇关于合同解释原则的学术论文，
请：
1. 提取每篇的核心论点
2. 识别三篇文献的共同主张和分歧点
3. 生成一个300字的文献综述段落，帮我定位我的研究立场
```

#### 第三层：案例分析
```
我有以下10份裁判文书，请分析：
1. 各案法院对"情势变更"的认定标准有何异同？
2. 支持适用情势变更的案件有哪些共同特征？
3. 裁判结果与合同约定之间的关系模式？
```

#### 第四层：写作辅助
```
基于我们的讨论，请帮我起草论文"数据可携权的中国实践路径"
的第二节"现行法律框架"部分，约2000字，
要求：学术风格、引用格式为中国法学期刊通用格式、
结尾与下一节"制度完善建议"形成逻辑衔接。
```

## 产品选型

### 主流法律 AI 产品横向对比

| 产品 | 定位 | 优势 | 局限 | 费用 |
|------|------|------|------|------|
| **Claude（Anthropic）** | 通用 AI 助手 | 长文本处理、逻辑分析强 | 不联网（需 MCP） | 订阅制 |
| **ChatGPT（OpenAI）** | 通用 AI 助手 | 生态丰富、插件多 | — | 订阅制 |
| **通义千问（阿里）** | 通用 AI | 中文理解优秀 | — | 部分免费 |
| **文心一言（百度）** | 通用 AI | 中文场景、百度生态 | — | 部分免费 |
| **法狗狗** | 法律专用 | 裁判文书检索专注 | 功能较垂直 | 付费 |
| **幂律智能** | 法律专用 | 合同分析专业 | 价格较高 | 付费 |
| **ChatLaw** | 学术法律 AI | 开源、学术研究 | 产品化程度低 | 开源 |

### 选型框架

**第一步：明确使用场景**
- 合同审查 → 文本分析能力为主
- 法律研究 → 知识覆盖 + 数据库接入
- 客户咨询 → 中文表达 + 安全合规

**第二步：评估数据安全**
- 你的数据是否会被用于训练模型？
- 是否有企业版（数据隔离）？
- 是否符合你所在机构的 IT 安全政策？

**第三步：小范围试用**
- 选择2-3个实际案例作为测试集
- 评估输出质量、稳定性和易用性

**第四步：考虑整合成本**
- 是否有 API？
- 是否可接入现有工作系统（文档管理、案管系统）？
