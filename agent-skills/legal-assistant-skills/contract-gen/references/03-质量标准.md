# 合同生成质量自检清单

合同生成完毕后，按以下四维度逐项自检，确保输出质量。

---

## 一、结构完整性

### 必备结构检查
- [ ] 合同标题准确反映合同类型
- [ ] 主体信息完整（名称、证件号、联系方式、地址）
- [ ] 根据合同类型包含了所有必备条款
- [ ] 根据合同类型删除了不适用的条款
- [ ] 条款编号连续，无跳号、重号
- [ ] 签字盖章区域完整，预留足够空间
- [ ] 有生效条款（合同份数、生效条件）

### 条款逻辑检查
- [ ] 条款排列顺序合理（主体 → 标的 → 价款 → 履行 → 权利义务 → 违约 → 争议解决 → 生效）
- [ ] 无多余或重复的条款
- [ ] 每个条款内容充实，非空壳条款

---

## 二、内容一致性

### 术语和指代一致
- [ ] 同一概念在全文使用相同名称（如"标的物"不混用"产品""货物"）
- [ ] 当事方称谓前后一致（如"甲方""买受方"不混用）
- [ ] 名词解释中定义的术语在全文统一使用

### 数字和金额一致
- [ ] 金额大小写一致
- [ ] 百分比数字前后引用一致
- [ ] 期限日期在不同条款中不矛盾
- [ ] 用户未提供的具体数字使用 `[____]` 占位符

### 逻辑一致
- [ ] 付款节点与履行进度匹配
- [ ] 违约责任与对应义务匹配
- [ ] 解除条件与违约行为定义匹配
- [ ] 保密期限与合同期限的关系合理
- [ ] 质保期与验收时间的衔接合理
- [ ] 不同条款之间无逻辑矛盾

### 交叉引用一致
- [ ] 条款之间的交叉引用正确（如"依据本合同第X条"）
- [ ] 附件引用与实际附件对应

---

## 三、立场一致性

### 整体立场检查
- [ ] 违约责任条款体现了用户选择的立场倾斜
- [ ] 付款条件体现了用户选择的立场倾斜
- [ ] 风险分配体现了用户选择的立场倾斜
- [ ] 争议管辖体现了用户选择的立场倾斜
- [ ] 倾斜程度合理，不构成显失公平

### 合法性检查
- [ ] 立场倾斜未违反法律强制性规定
- [ ] 劳动合同不违反劳动法对劳动者的强制保护
- [ ] 格式条款未排除对方主要权利或免除己方主要责任
- [ ] 违约金比例在司法实践可支持的范围内

---

## 四、可执行性

### 条款可操作性
- [ ] 违约金有明确的计算基础和计算方式（如"按未付金额的日万分之五计算"）
- [ ] 通知义务有具体的时间限制（如"在事件发生之日起 3 日内"而非"及时通知"）
- [ ] 验收标准具有可操作性（有明确的检验方法和判定标准）
- [ ] 解除合同的触发条件可客观判定

### 法律效力
- [ ] 约定的仲裁机构真实存在且名称准确
- [ ] 约定的管辖法院具有管辖权（与合同签订地、履行地、被告住所地等有关联）
- [ ] 不存在仲裁和诉讼同时约定的冲突
- [ ] 违约金比例在司法实践可支持的范围内（一般不超过实际损失的 30%）
- [ ] 无流质/流押等法律明确禁止的条款

### 法律引用准确性
- [ ] 引用的法律法规名称准确（如《中华人民共和国民法典》）
- [ ] 引用的条款号与实际条文内容对应
- [ ] 引用的法律法规未被废止或修订

---

## 五、格式规范性

### Markdown 格式
- [ ] 使用一级标题（#）作为合同标题
- [ ] 使用二级标题（##）作为章节标题
- [ ] 使用有序列表或层级编号作为条款编号
- [ ] 金额、日期等关键信息使用加粗标记
- [ ] 占位符使用 `[____]` 格式
- [ ] 不使用代码块标记包裹合同正文

### 专业规范
- [ ] 使用准确的法律术语
- [ ] 避免"合理时间""适当方式"等模糊表述（如无法避免则加以限定）
- [ ] 法律依据引用准确（如《民法典》条款号）
- [ ] 日期格式统一（如"XXXX年XX月XX日"）
- [ ] 货币表述规范（如"人民币（大写）XX元整"）

### 输出规范
- [ ] 仅输出合同正文，不含说明性前后缀
- [ ] 不含致谢、提示语、调试信息
- [ ] 末尾附免责声明（与合同正文分隔）
